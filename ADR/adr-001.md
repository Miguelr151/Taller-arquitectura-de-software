ADR-001: Descomposición del Monolito en Microservicios
Contexto

El sistema actual "Espagueti de Encuestas" está construido bajo una arquitectura monolítica donde el controlador concentra lógica de negocio, acceso a datos y validaciones, violando el principio de responsabilidad única (SRP). Esto dificulta la escalabilidad y mantenimiento.

Decisión

Se decidió dividir el sistema en los siguientes bounded contexts:

survey-service → Gestión de encuestas

voting-service → Gestión de votos

API Gateway → Punto único de entrada

Consecuencias

Positivas:

Escalabilidad independiente

Despliegue desacoplado

Mejor mantenibilidad

Negativas:

Mayor complejidad operativa

Comunicación entre servicios

Alternativas consideradas

Mantener monolito modular → descartado por baja escalabilidad.

Arquitectura hexagonal dentro del monolito → descartada por no cumplir objetivo de desacoplamiento total.
