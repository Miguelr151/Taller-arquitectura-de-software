ADR-002: Uso de Spring Data JPA como estrategia de persistencia
Contexto

El sistema actual utiliza SQL manual concatenado directamente en el controlador, lo que genera riesgo de SQL Injection y duplicación de código.

Decisión

Se adopta Spring Data JPA como capa de persistencia estándar para los microservicios.

Consecuencias

Positivas:

Seguridad frente a inyección SQL

Reducción de código repetitivo

Integración nativa con Spring

Negativas:

Curva de aprendizaje inicial

Dependencia del ORM

Alternativas consideradas

JdbcTemplate → descartado por menor nivel de abstracción.

MyBatis → descartado por no ser estándar del ecosistema actual del proyecto.
